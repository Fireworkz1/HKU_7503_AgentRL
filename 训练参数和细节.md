# 训练参数与配置

## DPO v2（训练有效）

```
命令: python train_dpo.py --epochs 2 --batch_size 4 --learning_rate 5e-7 --log_interval 50 --save_interval 2000 --from_weight full_sft --data_path ../dataset/dpo.jsonl --max_seq_len 1024 --device cuda:0 --save_weight dpo_v2
```

| 参数 | 值 |
|------|-----|
| 基础权重 | full_sft_768.pth |
| 训练数据 | dpo.jsonl（17166 条偏好对） |
| Epochs | 2 |
| Batch Size | 4 |
| Learning Rate | 5e-7（cosine decay） |
| Max Seq Len | 1024 |
| Beta (KL系数) | 0.15 |
| 梯度裁剪 | 1.0 |
| 混合精度 | bfloat16 |
| Reward Model | 无需 |
| GPU | RTX 4090 × 1 |
| 训练时间 | ~14 min |
| 总步数 | 8584 (4292 × 2) |
| 模型参数量 | 63.912M (策略) + 63.912M (参考) |
| 结果 | loss 0.5403 → 0.1272，下降 76%，收敛正常 |

---

## GRPO（规则奖励）

```
命令: python train_grpo.py --epochs 1 --batch_size 4 --learning_rate 5e-6 --from_weight full_sft --data_path ../dataset/rlaif.jsonl --rollout_engine torch --thinking_ratio 0.1 --device cuda:0
```

| 参数 | 值 |
|------|-----|
| 基础权重 | full_sft_768.pth |
| 训练数据 | rlaif.jsonl |
| Epochs | 1 |
| Batch Size | 4 |
| Learning Rate | 5e-6 |
| Rollout Engine | torch |
| Thinking Ratio | 0.1 |
| Num Generations | 4（每个 prompt 生成 4 个回答） |
| Epsilon (clip) | 0.2 |
| Beta (KL系数) | 0.1 |
| Reward Model | 无（规则奖励） |
| 规则奖励 | 长度分(±0.5) + Think格式(±1.0) + 闭合分(±0.25) + 重复惩罚 |
| GPU | RTX 4090 × 2 |
| 训练时间 | ~1 h |
| 总步数 | 9751 |
| 模型参数量 | 63.912M (策略) + 63.912M (参考) |
| 结果 | reward 0.2048 → 1.3687，训练有效，但出现 reward hacking |

### GRPO 训练日志关键指标

```
Step 5000: Reward=1.20,   KL_ref=...,   Adv_Std=...,  Actor_Loss=...
Step 9751: Reward=1.3687, KL_ref=...,   Adv_Std=...,  Actor_Loss=...
```

## PPO（规则奖励）

```
命令: python train_ppo.py --epochs 1 --batch_size 4 --learning_rate 3e-7 --from_weight full_sft --data_path ../dataset/rlaif.jsonl --rollout_engine torch --device cuda:0
```

| 参数 | 值 |
|------|-----|
| 基础权重 | full_sft_768.pth |
| 训练数据 | rlaif.jsonl |
| Epochs | 1 |
| Batch Size | 4 |
| Actor Learning Rate | 3e-7 |
| Critic Learning Rate | 5e-7 |
| Rollout Engine | torch |
| Clip Epsilon | 0.2 |
| Cliprange Value | 0.2 |
| KL Coef | 0.02 |
| Early Stop KL | 0.25 |
| Gamma | 1.0 |
| Lambda (GAE) | 0.95 |
| Reward Model | 无（规则奖励） |
| 规则奖励 | 同 GRPO |
| GPU | RTX 4090 × 2 |
| 训练时间 | ~1 h |
| 总步数 | 9751 |
| 模型参数量 | 63.912M (Actor) + 63.912M (Critic) + 63.912M (Ref) |
| 结果 | reward -0.25 → 0.2083，训练基本失败 |

### PPO 训练日志关键指标
```
Step 1:    Reward=-0.25, KL_ref=2.65,  Approx_KL=0.30, ClipFrac=0.23, Critic_Loss=0.35
Step 10:   Reward=0.2083, KL_ref=0.004, Approx_KL=0.001, ClipFrac=0.00, Critic_Loss=0.16
Step 9751: Reward≈0.2083, 几乎不再变化
```

## 四种方法参数对比

| 参数 | DPO v2 | GRPO | PPO | 说明 |
|------|--------|------|-----|------|
| 网络数量 | 2 (策略+ref) | 2 (策略+ref) | 3 (actor+critic+ref) | PPO 多一个 Critic |
| 显存占用 | ~6 GB | ~10 GB | ~14 GB | PPO 最重 |
| 需要 RM | 否 | 是 | 是 | DPO 最简单 |
| 需要 Rollout | 否 | 是 | 是 | DPO 离线训练 |
| 学习率 | 5e-7 | 5e-6 | 3e-7 / 5e-7 | DPO 需要更低 lr |
| 训练数据 | 偏好对 | prompt-only | prompt-only | DPO 数据成本更高 |
| 训练时间 | 14 min | 1 h | 1 h | DPO 最快 |
