### PPO "平衡平衡" 的原因
PPO 在非 tool_call 场景下的 reward 信号主要来自：
1. 长度分：5-800字符 → +0.5（重复"平衡"轻松满足）
2. 思考长度分：20-300字符 → +1.0（重复填充即可）
3. 思考闭合分：正确闭合 </think> → +0.25
4. RM 打分：InternLM2-1.8B-Reward 对"平衡/确保/共融"等词给正分

模型找到了一条捷径：输出重复的"高级词汇"即可稳定拿到正 reward，无需真正推理。
这是典型的 reward hacking / mode collapse。

### 结论
64M 参数量的模型推理能力本身极弱，所有权重在 thinking 场景下都无法给出正确答案。
但 RL 训练显著提升了 Think 标签的**格式可靠性**（即能否正常输出 `<think>...</think>` 标签）：
- SFT：40%（30题中仅12题正常输出）
- GRPO：86.7%（26/30）
- DPO：90%（27/30）
